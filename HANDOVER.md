## 1. プロジェクト概要

### 目的
drawDB-io/drawdb を Tauri 2.0 でデスクトップアプリ化した khsuzan/drawDB-App をベースに、以下を完全実装するオーバーレイ。

### ライセンス
- ベース drawDB: AGPL-3.0
- 本オーバーレイ: 同じく AGPL-3.0 で公開すること
- 再配布時は派生ソース公開義務あり（社内利用なら不要）

### スタック
| レイヤ | 採用 |
|---|---|
| GUI フレームワーク | Tauri 2.0 (WebView2/WKWebView/WebKitGTK) |
| フロント | React + Vite (drawDB 既存をそのまま) |
| バックエンド言語 | Rust (single-instance/dialog/fs/sql) |
| ローカル永続化 | IndexedDB (Dexie, 既存) + SQLite (オプション, tauri-plugin-sql) |
| ファイル形式 | .ddb (JSON), .ddbpack (ZIP), .xlsx (Excel), .sql (Oracle/MySQL) |

## 8. 設計判断ログ

### 8-1. なぜ Tauri か（Dioxus 不採用理由）
drawDB は React + Vite + SVG キャンバスで書かれており、Dioxus（純Rust GUI）にすると全画面の書き直しが必要。Tauri ならフロントをそのまま流用できる。

### 8-2. なぜ single-instance + argv か（deep-link 不採用理由）
Tauri の fileAssociations は、Win/Linux で tauri-plugin-deep-link の onOpenUrl が発火しない既知バグ (tauri-apps/plugins-workspace#1990 / tauri-apps/tauri#11557) があるため、single-instance の argv 経由で .ddb/.ddbpack/.xlsx を検出して emit("open-file") する方式を採用。

### 8-3. なぜ IndexedDB を残したか
既存 drawDB の全画面・テンプレ・Open/Save UI が Dexie に直結しており、SQLite に完全置換すると改修範囲が膨大。よって：
- 既定: IndexedDB のまま（ブラウザ版とコード共通）
- オプション: SQLite を併用（永続化先を選べる）

### 8-4. なぜ .ddbpack を JSON でなく ZIP にしたか
- 複数ダイアグラム + テンプレ + 将来のアセット（画像/添付）を1ファイルにまとめる
- 各 .ddb は単体でも開けるので部分復旧しやすい
- manifest.json で索引、破損時に診断容易

### 8-5. なぜ Excel 型推論をサンプル500行に制限したか
- Oracle/MySQL のテーブル設計はサンプル500行で型分布がほぼ収束
- 100万行のExcelでも 500ms 以内で完了
- 全数走査だと WebView が固まる

### 8-6. なぜ SmartScreen 警告対策を含めなかったか
コード署名証明書（DigiCert/SSL.com で $200〜$700/年）が必要で、鍵を作れる立場でないとこのチャットでは完結不可。タスクリスト T1〜T3 として残置。

## 10. トラブルシューティング

### Q1. npm run tauri build で WiX が落ちる
- VBSCRIPT optional feature を有効化（Settings → Apps → Optional features → More Windows features）
- または tauri.conf.json から "msi" ターゲットを削除し "nsis" のみにする

### Q2. SmartScreen 警告が出る
- 未署名のため正常。「詳細情報」→「実行」で起動可
- 解消には T1〜T2 の証明書購入が必要

### Q3. .ddb をダブルクリックしても drawDB が起動しない
- インストーラを管理者権限で再実行してファイル関連付けを再登録
- HKCU\Software\Classes\.ddb と .ddbpack を確認

### Q4. Excel インポートで型推論がおかしい
- 1行目をヘッダ行にしているか確認
- データ部の先頭500行内に NULL/混在型がないか確認
- どうしても合わない場合は drawDB エクスポート形式に従って手動で整形

### Q5. Oracle DDL のパースで FK が抜ける
- ALTER TABLE ... ADD CONSTRAINT ... FOREIGN KEY ... REFERENCES ...; 形式が前提
- インラインFK は未対応（v2.1 で対応予定）

### Q6. npm run tauri build が遅い
- 初回のみ Rust の依存ビルドで 10〜20分
- 2回目以降は1〜3分
- lto = true を [profile.release] から外せば 30〜50% 高速化（ただし exe サイズ増）
